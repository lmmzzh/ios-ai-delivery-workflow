# Plan Template

## 1. Background

- What is the current task?
- Why does it need to be done now?
- What is the surrounding project context?

## 2. Goal

- What is the main delivery goal of this round?
- What is the smallest acceptable outcome?
- What user-visible change should happen after completion?

## 3. Scope

- What is allowed to change in this round?
- What is explicitly out of scope?
- Where does this round connect with the existing stable flow?

## 4. Current Problem

- What are the main issues in the current implementation?
- Which issues are the real target of this round?
- Which issues exist but should not be treated as the main target now?

## 5. Proposed Solution

- What solution is recommended?
- Why is this solution preferred?
- If there are alternatives, why are they not preferred now?

## 6. Risk Control

- What are the main risks?
- How will the change radius be limited?
- What should be validated carefully or kept rollbackable?

## 7. Implementation Order

- What should be done first?
- What should be done next?
- At which point should validation happen before continuing?

## 8. Acceptance

- What counts as done for this round?
- Which key paths must be verified?
- Which results would indicate the plan should be re-evaluated?
