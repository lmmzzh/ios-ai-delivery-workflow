# Notes

This directory contains short, desensitized notes extracted from real delivery work.

The purpose is not to dump internal project material.

The purpose is to preserve reusable decisions, examples, and workflow patterns that can safely be made public.

## Current Notes

### display-layer-handoff-case.md

A desensitized handoff example showing how one round of display-layer work can be documented without turning into a long diary.

It is useful if you want to see:

- what a real boundary-aware handoff looks like
- how "what was not changed" can be written clearly
- how a task can stay in the display layer without spreading into the core flow

## Principles

- desensitize first
- keep the engineering judgment
- preserve boundaries, not internal identifiers
