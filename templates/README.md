# Templates

This directory contains reusable templates extracted from real project delivery work.

The goal is not to create pretty documents.

The goal is to reduce repeated thinking, preserve working context, and make future iterations easier to continue.

## Current Templates

### plan-template.md

Use this when a task should be analyzed before implementation.

Best for:

- new feature access
- legacy project iteration
- medium-sized refactors with clear boundaries

### handoff-template.md

Use this when one round of work should be handed off cleanly to the next round.

Best for:

- daily progress handoff
- multi-round collaboration
- AI-assisted development where context is easy to lose

### collaboration-memory-template.md

Use this when a project starts accumulating rules and stable context that should not be explained again and again.

Best for:

- longer-running projects
- repeated collaboration with AI
- preserving boundaries, naming rules, and default decisions

## Principles

- clear boundaries over long notes
- reusable structure over one-off summaries
- stable context over fragile memory
